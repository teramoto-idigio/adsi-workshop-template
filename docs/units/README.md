# Unit of Work — 依存図と Phase

## 依存図

```
Unit 0: 共通基盤（プロジェクト骨格・認証・マスタ）
    │
    ├─────────────────────────┐
    ▼                         ▼
Unit 1: 勤怠ドメイン        Unit 2: 申請・承認ドメイン
(勤怠入力/カレンダー/集計/CSV)  (有給/残業/承認/CSV)
    │                         │
    └─────────┬───────────────┘
              ▼
        統合テスト（全体結合）
```

## Phase 割り当て

| Phase | Unit | 担当 | 内容 |
|-------|------|------|------|
| A | Unit 0 | 2人共同 | プロジェクト生成・共通基盤・認証・マスタ管理 |
| B | Unit 1 | 担当者① | 勤怠記録・カレンダー・集計・個人CSV |
| B | Unit 2 | 担当者② | 有給申請・残業申請・承認・部署CSV |
| C | — | 2人共同 | 統合テスト・E2E |

## 並行実装の前提

Unit 0 で以下を確定するため、Unit 1 / Unit 2 は互いに依存なく並列実装可能:

- Flyway マイグレーション（全テーブル）
- Entity / Enum / 共通 DTO
- Security 設定（認証・認可フィルタ）
- 共通例外ハンドラ
- テスト基盤（テスト用プロファイル・ベースクラス）

## 担当の目安

- **担当者①（Unit 1）**: 勤務時間計算ロジック（深夜・残業・フレックス清算）が中心。計算の正確性がポイント。
- **担当者②（Unit 2）**: 申請→承認のステートマシンが中心。権限チェックとステータス遷移がポイント。
