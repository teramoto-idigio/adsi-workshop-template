# 勤怠管理アプリ — 要求仕様書

## 概要

企業の従業員（約100名規模）が日々の勤怠を記録・管理し、マネージャーが部署メンバーの勤怠を確認・承認するWebアプリケーション。

## 技術スタック

- Backend: Java / Spring Boot
- Frontend: Next.js / TypeScript
- DB: PostgreSQL（想定）
- 認証: ID / パスワード認証

---

## ユーザーロールと権限

| ロール | 権限 |
|--------|------|
| 一般社員 | 自分の勤怠入力・閲覧、有給申請、残業申請、自分の月次一覧CSV出力 |
| マネージャー | 上記 + 自部署メンバーの勤怠閲覧、有給/残業申請の承認、部署月次サマリーCSV出力 |
| 管理者 | 上記 + 全社員の勤怠閲覧、部署管理、社員管理、有給付与 |

---

## ユーザーストーリー

### US-1: 勤怠入力

> 一般社員として、日々の出勤時刻・退勤時刻・休憩時間を手入力したい。正確な勤務記録を残すために。

**受け入れ基準:**
- 出勤時刻、退勤時刻、休憩時間（分単位）、備考を入力できる
- 休憩時間は手動入力（自動計算なし）
- 入力後に勤務時間（退勤 − 出勤 − 休憩）が自動表示される
- 過去日の入力・修正も可能（承認フロー不要）

### US-2: 月次勤怠カレンダー表示

> 一般社員として、自分の月次勤怠をカレンダー形式で一覧したい。出勤状況を把握するために。

**受け入れ基準:**
- カレンダー形式で1ヶ月分の勤怠を表示
- 各日に出勤時刻・退勤時刻・勤務時間を表示
- 有給・半休の日は視覚的に区別できる
- 月の集計（総労働時間、残業時間、深夜勤務時間）をサマリー表示

### US-3: 勤務時間集計

> 一般社員として、月次の総労働時間・残業時間・深夜勤務時間を確認したい。自分の勤務状況を把握するために。

**受け入れ基準:**
- 日次集計: 勤務時間、残業時間（8.5時間超過分）、深夜勤務時間（22:00〜5:00）
- 月次集計: 総労働時間、総残業時間、総深夜勤務時間
- フレックス清算: 月の所定労働時間との過不足を表示

### US-4: 有給休暇申請

> 一般社員として、有給休暇（全休・AM半休・PM半休）を申請したい。休暇を取得するために。

**受け入れ基準:**
- 全休、AM半休、PM半休を選択して申請できる
- 申請時に有給残日数が表示される
- 残日数が0の場合は申請できない（エラー表示）
- マネージャーに承認依頼が通知される
- 承認/却下の結果が申請者に通知される

### US-5: 有給残日数管理

> 管理者として、社員の有給休暇日数を付与・管理したい。法令に基づく有給管理を行うために。

**受け入れ基準:**
- 管理者が社員に有給日数を付与できる
- 消化日数と残日数が自動計算される（半休は0.5日消化）
- 社員ごとの有給付与・消化履歴を確認できる

### US-6: 残業申請

> 一般社員として、残業を事前または事後に申請したい。残業の記録と承認を得るために。

**受け入れ基準:**
- 事前申請: 予定残業時間と理由を入力して申請
- 事後申請: 実績残業時間と理由を入力して申請
- マネージャーに承認依頼が通知される
- 承認/却下の結果が申請者に通知される

### US-7: 申請承認（マネージャー）

> マネージャーとして、部下の有給申請・残業申請を承認/却下したい。部署の勤怠を管理するために。

**受け入れ基準:**
- 未承認の申請一覧を確認できる
- 申請内容を確認し、承認または却下（却下理由入力）できる
- 自部署のメンバーの申請のみ表示される

### US-8: 部署メンバー勤怠閲覧（マネージャー）

> マネージャーとして、自部署メンバーの勤怠状況を一覧で確認したい。部署の労務状況を把握するために。

**受け入れ基準:**
- 自部署メンバーの当月勤怠サマリーを一覧表示
- 各メンバーの総労働時間、残業時間を確認できる
- メンバーを選択して日別の勤怠詳細を確認できる

### US-9: CSV エクスポート

> 一般社員として自分の月次勤怠一覧を、マネージャーとして部署全員の月次サマリーをCSV出力したい。記録を外部で活用するために。

**受け入れ基準:**
- 一般社員: 自分の月次勤怠一覧（日別の出退勤・勤務時間）をCSVダウンロード
- マネージャー: 部署全員の月次サマリー（社員名、総労働時間、残業時間等）をCSVダウンロード
- 対象月を選択可能

### US-10: 社員管理（管理者）

> 管理者として、社員の登録・編集・部署割り当てを行いたい。組織の人員を管理するために。

**受け入れ基準:**
- 社員の新規登録（氏名、メールアドレス、ロール、部署）
- 社員情報の編集
- 部署異動: ワンクリックで所属部署を変更可能
- 社員の無効化（論理削除）

### US-11: 部署管理（管理者）

> 管理者として、部署の作成・編集を行いたい。組織構造を管理するために。

**受け入れ基準:**
- 部署の新規作成（部署名）
- 部署名の編集
- 部署の削除（所属社員がいない場合のみ）
- フラット構成（階層なし）

### US-12: 認証

> ユーザーとして、ID/パスワードでログインしたい。セキュアにアプリを利用するために。

**受け入れ基準:**
- メールアドレス + パスワードでログイン
- パスワードはハッシュ化して保存
- ログアウト機能
- 未認証時はログイン画面にリダイレクト

## フレックスタイム制度の仕様

| 項目 | 値 |
|------|-----|
| コアタイム | 9:00〜17:30 |
| 1日の所定労働時間 | 8.5時間 |
| 清算期間 | 1ヶ月 |
| 残業の定義 | 1日8.5時間超過分 |
| 深夜勤務 | 22:00〜5:00 |

---

## 非機能要件

| 項目 | 要件 |
|------|------|
| 想定ユーザー数 | 約100名 |
| 同時アクセスピーク | 9:00、17:30（始業・終業時間帯） |
| データ保持期間 | 3年（労基法準拠） |
| モバイル対応 | 不要（デスクトップブラウザのみ） |
| 外部システム連携 | なし |
